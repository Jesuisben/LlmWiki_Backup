# 배열 vs 컬렉션

## 비교 목적

Java 수업에서는 먼저 배열을 배우고, 총정리에서 컬렉션으로 이어질 가능성을 남겼다. 두 자료구조는 여러 값을 담는다는 점은 같지만 크기와 기능에서 차이가 크다.

## 한눈에 보기

| 항목 | 배열 | 컬렉션 |
|---|---|---|
| 크기 | 생성 시 고정 | 보통 동적으로 변경 |
| 문법 예 | `int[] arr = new int[3]` | `List<String> list = new ArrayList<>()` |
| 인덱스 | 0부터 시작 | List는 인덱스, Set/Map은 구조별로 다름 |
| 수업 사용 | 점수, 이름, 객체 배열 | 이후 Java/Spring에서 목록 데이터 처리 |
| 장점 | 단순하고 빠른 기본 구조 | 추가/삭제/검색 등 편의 기능 풍부 |

## 언제 무엇을 쓰는가

1. **BTS 멤버처럼 개수와 값이 정해진 목록:** `String[]` 초기화 기법과 일반/향상된 `for`로 전체 조회하는 배열 실습이 적합하다.
2. **상품 객체 두 개처럼 고정 크기 실습:** `Product03[] itemlist = new Product03[2]`로 칸을 먼저 만들고 각 칸에 객체를 연결한다.
3. **추가·삭제가 반복되는 목록:** 숙제에서 조사한 `List`처럼 크기가 달라지고 `add`, `remove`, `size`가 필요한 경우 컬렉션이 더 자연스럽다.

## 관련 수업 예시와 범위

- **Java 수업에서 직접 실행:** 기본형 배열, 문자열 배열, 객체 배열, 일반/향상된 `for` 순회.
- **숙제로 조사:** `List`, `Set`, `Map`의 순서·중복·주요 메서드·구현 클래스. 이 과목 raw에는 컬렉션을 사용한 본 수업 실행 코드보다 조사 답안이 중심이다.
- **이후 확장:** Spring에서 조회 결과 목록을 컬렉션으로 다루는 관점은 후속 응용이며 Java 날짜 수업의 직접 구현으로 간주하지 않는다.

## 헷갈리기 쉬운 포인트

- 배열은 `length` 속성을 쓰고, 컬렉션은 보통 `size()` 메서드를 쓴다.
- 배열은 기본 문법이고, 컬렉션은 Java 라이브러리의 자료구조다.
- 객체 배열은 배열이지만, 각 칸에는 객체 참조가 들어간다는 점에서 참조 자료형 이해가 필요하다.
- `Set`과 `Map`은 `List`처럼 “인덱스로 꺼내는 가변 배열”이 아니다. 컬렉션 전체를 배열의 확장판 하나로만 이해하면 순서·중복·키 구조 차이를 놓친다.
