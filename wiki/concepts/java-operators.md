# Java 연산자

## 정의

연산자는 변수나 값에 대해 계산, 비교, 대입, 논리 판단을 수행하는 기호다. 수업에서는 산술 연산자에서 시작해 대입, 복합 대입, 증감, 비교, 논리, 삼항 연산자로 확장했다.

## 왜 중요한가

Java 프로그램은 값을 저장한 뒤 연산하고, 그 결과로 조건을 판단한다. 따라서 연산자는 [[concepts/java-conditional-logic|조건 판단]]과 [[concepts/java-loop|반복문]]의 기반이다.

## 핵심 설명

| 분류 | 예 | 수업 맥락 |
|---|---|---|
| 산술 | `+`, `-`, `*`, `/`, `%` | 합계, 평균, 홀짝 판단 |
| 대입 | `=` | 변수에 값 저장 |
| 복합 대입 | `+=`, `-=`, `*=`, `/=`, `%=` | 기존 값에 누적 계산 후 저장 |
| 증감 | `++`, `--` | 1씩 증가/감소, 전위/후위 차이 |
| 비교 | `<`, `<=`, `>`, `>=`, `==`, `!=` | 결과는 `boolean` |
| 논리 | `&&`, `||`, `!` | 여러 조건 결합/부정 |
| 조건 | `조건 ? 참 : 거짓` | 삼항 연산자 |

## 수업 예시

```java
int a = 3, b = 4, c = 5;
int result = 2 * a + 3 * b - c; // 13
```

`2026.02.27` 문제 이미지는 위처럼 산술 연산과 출력 형식을 연결했다. `2026.03.03` 마무리 문제는 복합 대입, 문자열 연결, 비교 연산을 한꺼번에 확인하는 역할을 했다.

```java
int x = 3, y = 5, z;
z = x++ + --y;          // x=4, y=4, z=7
z += --x + y++;         // x=3, y=5, z=14

double average = (double) 14 / 5; // 2.8
String parity = x % 2 == 0 ? "짝수" : "홀수";
```

연산식은 전위 증감 → 산술·비교 → 대입 → 후위 증감 완료 순으로 현재 값을 적어 가며 추적한다. 형변환은 나눗셈 전에 하느냐 뒤에 하느냐에 따라 소수 부분 보존 여부가 달라진다.

## 자주 헷갈리는 점

- `=`는 대입, `==`는 비교다.
- `x++`와 `++x`는 단독으로 쓰면 결과가 같아 보이지만, 다른 연산식 안에서는 값이 사용되는 시점이 다르다.
- 문자열과 `+`가 만나면 숫자 덧셈이 아니라 문자열 연결이 될 수 있다.
- `&&`는 둘 다 참이어야 참이고, `||`는 하나만 참이어도 참이다.
- Java에서 “같지 않음”은 `!=`다. 원본 일부의 `<>`는 Java 연산자가 아니다.
- `"합계: " + 1 + 2`는 왼쪽부터 문자열 연결이 진행되어 `합계: 12`가 된다. 숫자 합계를 먼저 원하면 괄호로 묶는다.
- `%`는 나머지 연산자다. 원본 일부의 `&` 표기는 같은 의미로 사용하지 않는다.
