- 숙제 01-01
클래스가 무엇이며, 객체는 무엇인가요?
	클래스 : 프로그램에 사용될 객체의 틀 정의, 객체 생성을 위한 설계도 역할
	-> 일종의 양식 저장소
	
\# - 클래스 : 기존 자료형을 이용하여 새롭게 만드는 사용자 정의 타입 (무한히 생성 가능)
-> 내가 만든 String2개와 int1개를 가진 새로운 형태의 데이터 타입인 Product

	객체 : 물리적으로 존재하거나 추상적으로 생각할 수 있는 것 중에 자신의 속성을
	가지고 있고, 다른 것과 식별 가능한 것
	(속성(필드)과 동작(메소드)으로 구성됨)


클래스와 객체는 어떤 연관성이 있나요?
	클래스에는 객체를 생성하기 위한 속성(필드)과 동작(메소드)이 정의되어 있다.
	클래스로부터 만들어진 객체를 클래스의 인스턴스라고 함. (인스턴스화 한다)
	main() 메소드가 없을 시 따로 객체 생성 과정을 거쳐서 사용해야 함.
	
클래스를 구성하는 3가지 요소는 무엇인가요?
	변수(속성)(필드)(Field), 동작(메소드)(Method), 생성자(Constructor)	
	생략되거나 복수 개가 작성될 수 있음
	
	속성 : 객체의 데이터가 저장되는 곳
	- 객체의 고유 데이터, 부품 객체, 상태 정보를 저장하는 곳
	선언된 형태는 변수와 비슷하지만, 필드를 변수라 부르지는 않음.
	변수는 생성자와 메소드 내에서만 사용 / 생성자와 메소드가 실행 종료되면 자동 소멸
	필드는 생성자와 메소드 전체에서 사용 / 객체가 소멸되지 않는 한 객체와 함께 존재
	
	동작 : 객체의 동작에 해당하는 실행 블록
	- 객체의 동작에 해당하는 중괄호 { } 블록
	중괄호 블록의 이름 = 메소드의 이름
	메소드 호출 시 중괄호 블록의 모든 코드들 일괄적 실행
	필드를 읽고 수정하는 역할도 하지만, 다른 객체를 생성해서 다양한 기능을 수행하기도 함
	객체 간의 데이터 전달 수단으로 사용됨
	외부로부터 매개값을 받을 수 있고, 실행 후 어떤 값을 반환할 수도 있음
	
	생성자 : 객체 생성 시 초기화 역할 담당
	- new 연산자로 호출되는 특별한 중괄호 { } 블록
	객체 생성 시 초기화 담당(필드 초기화, 메소드 호출하여 객체 사용 준비)
	메소드와 비슷하게 생겼지만, 클래스 이름으로 되어 있고, 반환 타입이 없음
	
변수(속성, property)와 메소드는 어떻게 다른 것인가요?
	변수 : 특정 시점의 상태를 저장해둔 정적인 개념 - 상태값을 가지고 있는 개념
	메소드 : 어떠한 역할, 기능을 수행하는 동적인 개념

- 숙제 01-02
자바의 static, stack, heap 메모리에 대하여 조사해 보세요.
1) static 메모리 (Method Area, Class Area, Code Area)
	필드 부분에서 선언된 전역변수와 정적 맴버 변수(static으로 선언되는 것)들의 데이터를
	메모리에 상주시킴

	전역변수 - 클래스 내부의 메소드 바깥에서 선언되어, 클래스 전체(혹은 다른 클래스에서도)
	자유롭게 접근 할 수 있는 변수 <-> 지역변수
	정적 맴버 변수 - 클래스라는 틀 자체에 붙어 있는 공용 변수
	프로그램 시작될 때 메모리에 자리를 잡고 프로그램이 끝날 때까지 그 자리에 고정됨.

2) stack 메모리
	기본 자료형을 생성할 때 저장하는 공간
	임시적으로 사용되는 변수나 정보들이 저장되는 영역
	참조 자료형은 heap 메모리에 존재하는 인스턴스(객체)만 저장함.

3) heap 메모리
	참조형(Reference Type)의 데이터 타입을 갖는 객체, 배열 등을 저장함
	new 키워드로 생성된 객체와 배열이 저장됨
	인스턴스(객체)의 실제 데이터는 heap 메모리에 올라감
	-> 저장된 메모리 위치가 다르기에 static 메서드에서 외부 인스턴스 맴버에 접근 X

	참조형 데이터 객체에 실제 데이터들이 담기는 공간
	-> 실제 데이터를 가지고 있는 heap 메모리 안의 참조값을 stack 메모리 안의 객체가 가지고 있음

	new를 통해 인스턴스 객체 생성 시 heap 영역에는 생성된 객체가 올라가고
	stack 영역에는 해당 객체에 대한 주소 값(Reference)이 저장됨

static 키워드는 언제 사용하면 좋을까요.
	- 모든 데이터가 공통된 데이터를 가져야 할때
	정적 변수가 필요할때
	- 객체의 속성과 상관없이 그저 입력받은 값으로만 계산해서 결과만 내놓을때
	- 상수(고정된 값)를 정의할 때

static 변수, 인스턴스 변수, 지역 변수의 각 특징은 어떠한 것이 있으며, 각각 어떠한 경우에 사용하나요.
1) static 변수
	- 특징
	클래스당 딱 하나만 생성되어 모든 객체가 공유함
	객체를 생성하지 않고도 바로 사용 가능 (클래스명.변수명)
	
	- 사용하는 경우
	모든 객체가 공통으로 가져야 하는 공유 데이터가 있을 때
	객체 생성 횟수를 세는 카운터나 시스템 설정값
	ex) 모든 사람의 고향, 원주율
	
2) 인스턴스 변수
	- 특징
	new를 통해 객체를 만들 때마다 각각의 객체 안에 독립적으로 생성됨
	객체마다 다른 값을 가질 수 있음
	
	- 사용하는 경우
	객체의 고유한 상태를 저장할 때
	객체가 살아있는 동안 데이터를 계속 유지해야 할 때
	ex) 시계의 생산지, 구동방식, 재질 등
	
3) 지역 변수
	- 특징
	메소드가 실행될 때 잠깐 생성된 후 메소드가 끝나면 즉시 사라짐
	반드시 초기화를 해야 사용할 수 있음
	
	- 사용하는 경우
	메소드 안에서 계산을 위한 임시 저장 공간이 필요할 때
	예시 : 반복문의 제어변수 i -> 그 반복문에서만 사용되고 그 메소드 밖에서는 사용 못함


- 숙제 01-03
접근 지정자(제어자)란 무엇이며, 이런 개념이 왜 필요하나요?
1) 접근 지정자(제어자)란 ?
	클래스, 변수, 메소드에 대한 접근 권한을 지정하는 것
	ex) 우리가 코드 작성할 때 보는 public

2) 필요한 이유
	당연하게도 해당 데이터들에 따라 접근 권한을 다르게 해야하기 때문에
	모두에게 보여주고 싶은 데이터들은 public으로
	잘못 건들이면 큰 문제가 생기는 데이터들은 private으로
	설정을 해야 하기 때문에
	(보안, 유지보수 때문)

getter(), setter()란 무엇인가요?
1) getter()
	private 접근 지정자가 붙은 인스턴스 변수(private변수) 값을 외부로 전달(반환)해주는 메소드

2) setter()
	외부에서 들어온 데이터 값을 검사한 뒤 private 접근 지정자가 붙은 인스턴스 변수(private변수)
	에 저장해주는 메소드

this 키워드란 무엇인가요?
	객체 내부에서 인스턴스 맴버에 접근하기 위한 키워드
	생성자와 메소드의 매개 변수 이름이 필드와 동일한 경우에
	인스턴스 맴버인 필드임을 명시할 때 사용

	객체가 new 연산자를 통해 메모리에 올라갈 때 가장 먼저 실행되는 특수한 메소드
	
2) 생성자의 필요성
	객체를 생성하자마자 기본 데이터인 초기값을 가지고 생성되게 하기 위함
	생성자가 없으면 객체를 만든 뒤 하나하나 객체의 값을 채워 넣어야하는 번거로움이 있음
	
3) 생성하는 방법
	조건 : 이름이 클래스 명과 동일 / 반환 타입(void, int 등)이 없음
	접근제어자 클래스이름(매개변수){
		
	}
	예시
	public homework(String name){
		this.name = name;
	}
	
생성자와 메소드의 차이는 무엇인가요.
	생성자는 객체가 생성될때 초기화를 시키고 메소드는 객체가 생성된 후 동작과 기능을 부여함
	생성자의 이름은 클래스 이름과 동일해야하지만 메소드는 아님
	생성자는 반환 타입이 없고 메소드는 반드시 명시해야 함
	생성자는 new로 딱 한 번 실행되고 메소드는 원할때 언제든 호출 가능함

숙제 02
상속이란 무엇인가요?
	부모 역할의 상위 객체가 자식 역할의 하위 객체에게 자기가 가지고 있는 필드와 메소드를
	하위 객체에게 물려주어, 하위 객체가 사용할 수 있도록 하는 것
	새로운 데이터 및 기능을 추가 가능함.
	
	상위 객체를 재사용 -> 하위 객체를 쉽고 빨리 설계
	(코드의 재사용성 높임, 반복된 코드 중복 줄임)
	
	상위 개체의 수정으로 모든 하위 객체들의 수정 효과를 가져오므로
	유지 보수 시간을 최소화 시킴 (유지보수성)
	
	ex) "빵"이라는 개념이 지구상에서 바뀌면
	그 하위 개념인 여러 종류의 빵을 다시 하나하나 재정의 하는 것이 아니라
	상위 개념인 "빵"을 재정의 하면 그 하위 개념인 여러 종류의 빵은 알아서 재정의가 됨
	
	일반적으로 상위 객체는 종류, 하위 객체는 구체적 사물
	ex) 상위 객체 : 시계, 하위 객체 : 기계식 시계, 전자 시계
	
상속 관계의 부모와 자식 클래스에서 생성자는 어느 쪽이 먼저 호출되는가요?
	부모 생성자 먼저 호출 후 자식 생성자가 호출됨
	
	자식 객체가 태어나려면 부모의 정보가 먼저 메모리에 세팅되어야 하기 때문

super 키워드는 누구를 지칭하는 건가요?
	부모 객체 자기 자신을 지칭
	this가 "나"를 가리킨다면
	super은 부모를 가리킴

오버라이딩이란 무엇이며, 이러한 개념이 왜 필요할까요.
1) 오버라이딩이란 무엇인가?
	부모에게 물려받은 메소드를 자식 클래스에 맞게 재정의하는 것
	위에서 언급했던 자식 클래스에 추가 수정 가능여부

2) 왜 필요한가?
	부모의 기능이 자식에게 100가지 맞아도 1가지는 맞지 않은게 있을 수 있고
	부모의 기능에는 없지만 자식에게는 필요한 것이 1개라도 있을 수 있어서

참조(레퍼런스) 형변환이 무엇인가요?
	객체의 타입을 부모나 자식 타입으로 바꾸는 것
	1) 업캐스팅(Upcasting) : 자식 객체를 부모 타입 변수에 담는 것 (암시적)
	2) 다운캐스팅(Downcasting) : 부모 타임에 담긴 객체를 다시 자식 타입으로 돌리는 것(명시적)
	
참조(레퍼런스) 형변환이 왜 필요할까요?
	(다형성)
	여러 종류의 자식 객체들을 하나의 부모 타입으로 묶어서 관리하기 위해서
	상속을 하는 이유와 동일하게 따로따로 관리하는 것보다 공통 분모를 하나로 관리하면
	코드가 간결해져서 유지보수에 좋기 때문에

instanceof 연산자는 어디에 활용이 되나요?
	객체가 특정 클래스의 인스턴스(진짜 정체)인지 확인할 때 사용

final 메소드란 무엇이며, 어떠한 경우에 사용할 수 있나요.
1) final 메소드랑 무엇인가?
	더 이상 오버라이딩(재정의)할 수 없는 메소드

2) 사용하는 경우
	보안을 걸거나 기능의 일관성 유지할 때

숙제 03
추상 클래스가 왜 필요한지 한번 조사해 보세요.
	공통적인 특징(어느정도의 가이드라인)을 모으지만
	구체적인 내용은 자식에게 맡기기 위해서 필요함

추상 클래스는 어떻게 만드나요?
	abstract 키워드 사용
	abstract class 클래스명 // 선언
	
	1) xx 하면 안됩니다.
		직적 객체 생성X
		{ } (몸통)을 가지면 안됨
	
	2) 반드시 yy해야 합니다.
		추상 메소드를 하나라도 포함하면 클래스에 반드시 abstract를 붙여야 함
		자식 클래스는 부모의 추상 메소드를 반드시 오버라이딩해야함
		
		* 반드시 세미콜론(;)으로 끝내야 함

다음 메소드들의 공통점과 차이점은 무엇인가요?

|            |  일반 메소드  |  final 메소드   |         추상 메소드         |
| ---------- | :------: | :----------: | :--------------------: |
| 구현부 ( {} ) |    O     |      O       |    X, 세미콜론(;)으로 끝남     |
| 오버라이딩      | 자유롭게 가능  |    절대 불가능    |        반드시 해야함         |
| 목적         | 공통 기능 제공 | 기능 변조 방지(보안) | 자식에게 구현 강제 (가이드라인만 제공) |


추상 클래스를 불완전 클래스라고 하는 이유는 무엇인가요?
	메소드 중에서 몸통 ( {} )이 없기 때문에
	정작 실행할 코드가 완성되어 있지 않아 아무런 동작을 할 수 없기 때문

인터페이스가 왜 필요한지 한번 조사해 보세요.
	서로 다른 클래스들에게 공통된 규칙(표준)을 적용하기 위해서
	상속은 A는 B다의 관계지만
	인터페이스는 A는 B기능을 할 수 있다.
	ex) 스마트폰과 컴퓨터는 부모가 다르지만 둘다 같은 충전기 규격을 지키면 사용가능

인터페이스는 어디에 사용할 수 있나요?
	표준화 : 여러 개발자 협업시 메소드 이름 통일
	다중 상속 효과 : 클래스 상속은 하나만 가능 / 인터페이스는 여러 개 구현 가능
	연결 고리 : 클래스 간의 결합도를 낮춰 하나를 바꿔도 다른쪽이 망가지지 않도록

인터페이스내의 변수와 메소드의 각 특징에 대하여 조사해 보세요.
	변수 : 반드시 상수(public static final)여야 함
	누구나 쓸 수 있고 변하지 않는 값만 가짐
	
	메소드 : 반드시 추상 메소드(public abstract)여야 함.
	몸통없이 이름만 존재 / 구현하는 클래스가 내용을 채워야 함
