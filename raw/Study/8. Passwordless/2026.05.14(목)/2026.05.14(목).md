#### \# 사이버 공격 4가지
1) 피싱 공격 - 40%


기법 : 이메일, 메시지 속임수
직원, 사용자 계정 -> 계정탈취 및 내부침투 목적
방어 방법 : Phishing-resistant MFA -> But 극복은 불가
피싱공격 : 사용자가 ID/PW 입력 -> 피싱공격 멀웨어가 설치된 PC를 해커가 들여다보고있음 (키 입력 탈취 - 스파이웨어)
-> 해커가 직접접속 or 우회접속


2) 멀웨어(랜섬)(랜섬웨어) - 30%


기법 : 악성코드 설치, 실행 (ex-USB 드라이브 (Auto Run))
개인PC, 서버, 의료장비 -> 데이터 암호화, 유출, 금전요구 목적
방어 방법 : 안티바이러스/EDR -> But 극복은 불가

- 랜섬웨어 공격용 멀웨어 : 이미 피싱공격을 당한 상태에서 시작됨
/ 이미 상대방에게 금전적 요구를 하면 받아들일정도로 PC에 중요한 스토리지가 있다고 파악해서 PC를 마비시킴


3) 소프트웨어 취약점(Remote Code Execution-원격코드실행) - 25% - 최근에 가장 부각되기 시작함

서버, 클라우드, 애플리케이션 -> 관리자 권한 탈취 및 내부접근 목적

#### \# 현재 보안 관련 기능들의 잘못된 토대
- Input-Based User Credential
사용자가 입력을 하기에 해커가 이 입력을 탈취하려고 기다림

- 잘못된 인식
1) 사용자는 정당한 사용자임을 입증할 책임이 있다.(무언가를 입력해야한다)


-> 사용자는 계정이 탈취되면 관리 부주의에 대한 책임이 있다.
-> 사용자 계정이 탈취되면 서비스 제공자는 책임을 지지 않는다.

2) 내 PC에는 악성코드가 있을 수 있다 (X)


-> 내 PC에는 악성코드가 반드시 있다 (O)



#### \# 패스워드리스
- 국제표준 번호부여되어 있음
ITU-T X.1280

- 개념
사용자 입증 책임을 온라인 시스템 입증책임으로 전환한다.

- 작동원리
아이디를 입력하면 사용자가 아니라 사용자가 사용하는 대상이 사용자에게 인증번호를 제시하고
모바일로 그 인증번호가 맞는지 체크함 (비밀번호 영역에서 사용자가 입력하는 것은 없음)
(오히려 서버가 사용자에게 입력을 함)(기존의 인증 방식의 정반대 방식)

- 기존 OTP와의 차이
기존 OTP는 어디선가 가져온 정보를 이용해서 만든 OTP를 다시 사용자가 입력하는것

- 경쟁사 대비 장점
1) 피싱, 파밍, 중간자 공격이 되지 않는 기술 (입력을 하지 않아서)


2) 사용자 인증과 서버 인증을 인증할때마다 검증함


3) 생체인증 센서를 추가할 수 없는 장치까지 지원하는 대역외 생체인증 기술


-> 별도의 센서가 필요없다는 전제하에 개발했기에 별도의 비용이 필요하지 않음
4) AI까지 지원하는 인증 기술


-> 프롬프트에서 패스워드리스로 인증하라고하면 패스워드리스로 인증시도하고 사용자가 인증하기를 기다림
AI agent를 이용할때 최종 결정권자를 사용자로 설정할 수 있는 장점이 있음

#### \# 인증 개념
신원인증 > 계정인증 > 연합인증

1) 신원인증


주요 기술 : 공동인증서 / 통신사본인인증 / 카카오인증 / 네이버인증 등
주요 사용처 : 계정생성 / 계정복구 / 계정로그인 / 자격확인 / 성년확인 등

2) 계정인증


주요 기술 : OTP / 생체인증 / FIDO / Passkey / 자동패스워드(X.1280)
주요 사용처 : 계정로그인

3) 연합인증


주요 기술 : OpenIDConnect / OAuth2 / SAML
주요 사용처 : 구글로그인 / Facebook 로그인 등 각종 소셜로그인
