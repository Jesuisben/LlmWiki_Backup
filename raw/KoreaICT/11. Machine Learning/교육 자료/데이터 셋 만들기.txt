좋은 질문입니다.

MNIST는 단순히 "이미지 파일들의 모음"이 아니라, 머신러닝이 학습하기 좋은 형태로 정리된 데이터셋입니다. 개발자가 직접 MNIST와 비슷한 데이터셋을 만드는 것도 충분히 가능합니다.

MNIST 데이터셋의 구조부터 이해하기

핵심 정리

MNIST와 같은 데이터셋은 특별한 형식이 아니라, **"입력 데이터(x)와 정답(y)를 학습용과 테스트용으로 나눈 NumPy 배열"**입니다. 따라서 개발자는 다음 순서로 직접 만들 수 있습니다.

데이터를 수집한다.
모든 데이터를 같은 크기와 형식으로 전처리한다.
각 데이터에 정답(Label)을 붙인다.
NumPy 배열로 변환한다.
train_test_split() 등으로 학습용과 테스트용으로 나눈다.
필요하다면 load_data() 함수를 만들어 MNIST처럼 (x_train, y_train), (x_test, y_test) 형태로 반환한다.

이 과정을 따르면 자신만의 MNIST와 유사한 데이터셋을 만들어 다양한 딥러닝 모델에 사용할 수 있습니다.

그럼 이것을 파일로 만들 수 있나요?

네, 파일로 만들 수 있습니다. 실제로 대부분의 공개 데이터셋도 내부적으로는 파일 형태로 저장되어 있습니다.

방법은 여러 가지가 있는데, 가장 많이 사용하는 방법을 설명드리겠습니다.

방법 1. NumPy 파일(.npz)로 저장 (가장 추천)

MNIST처럼 x_train, y_train 등을 한 번에 저장할 수 있습니다.

저장
import numpy as np

np.savez(
    "my_dataset.npz",
    x_train=x_train,
    y_train=y_train,
    x_test=x_test,
    y_test=y_test
)

그러면

my_dataset.npz

라는 파일이 생성됩니다.

불러오기
import numpy as np

data = np.load("my_dataset.npz")

x_train = data["x_train"]
y_train = data["y_train"]
x_test = data["x_test"]
y_test = data["y_test"]

사용 방법은 MNIST와 거의 같습니다.

방법 2. pickle 파일
import pickle

with open("dataset.pkl", "wb") as f:
    pickle.dump(
        (x_train, y_train, x_test, y_test),
        f
    )

읽을 때는

with open("dataset.pkl", "rb") as f:
    x_train, y_train, x_test, y_test = pickle.load(f)
방법 3. HDF5 파일

대용량 데이터에서는 많이 사용합니다.

import h5py

with h5py.File("dataset.h5", "w") as f:
    f.create_dataset("x_train", data=x_train)
    f.create_dataset("y_train", data=y_train)

    f.create_dataset("y_test", data=y_test)
방법 4. CSV 파일

이미지가 아닌 일반적인 표 형태의 데이터라면 CSV를 사용할 수도 있습니다.

예를 들어

age,height,weight,label
20,170,65,0
35,180,82,1
...

하지만 이미지 데이터에는 적합하지 않습니다.

MNIST처럼 load_data() 만들기

예를 들어

my_dataset.npz

를 만들어 놓았다면

import numpy as np

def load_data():
    data = np.load("my_dataset.npz")

    return (
        (data["x_train"], data["y_train"]),
        (data["x_test"], data["y_test"])
    )

사용은

(x_train, y_train), (x_test, y_test) = load_data()

처럼 MNIST와 완전히 동일합니다.

실제 Keras의 MNIST는 어떻게 저장되어 있을까?

Keras의 mnist.load_data()는 내부적으로 .npz 파일을 사용합니다.

예를 들어 처음 실행하면 인터넷에서 mnist.npz 파일을 다운로드하여 캐시에 저장하고, 이후에는 그 파일을 읽어서 다음과 같이 반환합니다.

(x_train, y_train), (x_test, y_test)

즉, Keras도 여러분이 만들 수 있는 것과 같은 방식으로 데이터를 저장하고 불러오는 것입니다.

가장 추천하는 방법

직접 데이터셋을 만든다면 .npz 파일을 사용하는 것을 추천합니다. 이유는 다음과 같습니다.

여러 개의 NumPy 배열을 하나의 파일에 저장할 수 있습니다.
저장과 불러오기가 매우 빠릅니다.
코드가 간단합니다.
mnist.load_data()와 유사한 방식으로 쉽게 사용할 수 있습니다.

따라서 사용자 정의 이미지 데이터셋을 만들 때는 .npz 파일 + load_data() 함수 조합이 가장 간편하고 실용적인 방법입니다.
