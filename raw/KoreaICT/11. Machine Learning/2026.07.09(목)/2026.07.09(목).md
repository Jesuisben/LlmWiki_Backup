- Class
머신 러닝에서 Class는 분류 모델이 예측하려는 각각의 범주(레이블)를 의미
\*\* 모든 Class는 범주형 데이터이지만, 모든 범주형 데이터가 Class인 것은 아님 \*\*
ex) 동물 -> 강아지, 고양이 등


- 혼돈 행렬 (머신 러닝 교안(이론).pdf(P. 197~207))
머신 러닝에서 가장 중요한? 개념

1) 개념
	혼돈 행렬(Confusion Matrix)는 머신 러닝 모델에 의해서 생성된
	분류 분석 모델의 성능을 지표화한 테이블

2) 항목

| 항목                | 설명                                                                      |
| ----------------- | ----------------------------------------------------------------------- |
| True/False        | 분류를 제대로 한 경우 True입니다.<br>모델이 정답을 잘 맞추었는지의 여부입니다.                        |
| Positive/Negative | 예측(분류) 결과가 Yes이면 Positive라고 표현합니다.<br>예측(분류) 결과가 No이면 Negative라고 표현합니다. |
-> 표 작성

|            | answer | Yes           | No            |
| ---------- | ------ | ------------- | ------------- |
| prediction |        |               |               |
| Yes        |        | 제대로 분류함(TP)   | 제대로 분류 못함(FP) |
| No         |        | 제대로 분류 못함(FN) | 제대로 분류함(TN)   |
예를 들어, FN은 실제 정답이 yes인데, no로 판단을 했다는 의미입니다.
즉, 정답을 못 맞췄다(F)이고, 예측치가 부정(N)이라는 의미입니다.
-> FP와 FN을 줄여야 함


3) 모델의 성능 평가에 사용 가능한 여러가지 지표들
	가장 일반적으로 많이 사용되는 항목은 Accuracy(정확도)

	3-1) 정확도(Accuracy) (머신 러닝 교안(이론).pdf(P. 200))
	(TN+TP)를 전체 관측치로 나눈 비율을 의미
	-> (TN+TP)/(TN+FP+FN+TP)
	
	3-2) 정밀도 (Precision) (머신 러닝 교안(이론).pdf(P. 201~202))
	분류 결과가 참인 항목 중에서 정말로 참으로 분류된 경우를 의미
	Positive라고 예측한 것 중 실제로 Positive인 비율
	-> TP/(FP+TP)
	
	3-3) 재현율(Recall) (TPR) (머신 러닝 교안(이론).pdf(P. 203~204))
	민감도 혹은 True Positive Rate라고도 함
	실제 정답이 참인 것 중에서 참으로 예측된 것들의 비율을 의미
	(실제 Positive인 것 중에서 Positive로 올바르게 예측한 비율)
	-> TP/(FN+TP)

	3-4) F1 Score (머신 러닝 교안(이론).pdf(P. 205~207))
	정밀도(Precision)와 재현율(Recall)의 균형을 나타내는 지표
	두 값을 조화 평균으로 계산



#### \# 분류 분석 모델 - SVM (머신 러닝 교안(이론).pdf(P. 221~230))
- 선형 분리(linear separable)
	임의의 공간에 직선을 그어서 서로 다른 분류를 분리시키는 것
	클래스를 구분하는 분리선과 가장 가까운 데이터 사이의 거리가 최대가 되는 분리선을 찾는다.

- SVM (Support Vector Machine) 개념
	(지도 학습 - 분류(Classification))
	데이터를 분류할 때 클래스 간의 경계를 가장 넓게 확보하는 초평면(Hyperplane)을 찾는 머신러닝 알고리즘
	-> 가장 최적화된 분리선을 생성하는 모델

- 용어 정리
	초평면 (Hyperplane) : 범주형 데이터를 분리하는 경계
	여백 (Margin) : 초평면과 가장 가까운 각 범주형 데이터 사이의 거리
	서포트 벡터 (Support Vectors) : 초평면과 가장 가까운 각 범주형 데이터 그룹에 속한 하나의 데이터
	커널 트릭 (Kernel Trick) : 커널 함수를 사용하여 비선형 문제를 선형 분류기로 해결할 수 있는 방법
	-> 분리하기 편하게 바꿔주는 것 (비선형을 선형으로 바꾸기)

```python

import seaborn as sns

# [Step 1] 데이터 준비 - Seaborn에서 제공하는 titanic 데이터 셋 가져 오기
print("# [Step 1] 데이터 준비 - Seaborn에서 제공하는 titanic 데이터 셋 가져 오기")
# load_dataset 함수를 사용하여 데이터프레임으로 변환
df = sns.load_dataset('titanic')

# 1은 생존자로써 342명이고, 0은 사망자로써 549명입니다.

print('# 중복이 되는 행은 제거하도록 합니다.')

print(df.info())

# 탑승 항구(embarked)와 도시 이름(embark_town) 컬럼은 다음과 같습니다.
# NaN 값이 많은 deck 열과, embarked과 데이터가 겹치는 embark_town 열은 삭제하도록 합니다.
print(df[['embarked', 'embark_town']].head(10))
rdf = df.drop(['deck', 'embark_town'], axis=1)

# how='any' 옵션은 해당 행 또는 열에 결측치가 하나라도 있으면 제거합니다.
rdf = rdf.dropna(subset=['age'], how='any', axis=0)

# 승선 도시(embarked) 컬럼은 범주형 데이터이므로, 범주 데이터 중에서 빈도 수가 가장 많은 항목으로 결측치를 치환하도록 합니다.
# mode() 메소드는 최빈도를 나타내는 항목을 찾아 주는 메소드입니다.
# mode()[0]는 최빈도수가 여러개 나오는 경우 1번째 항목을 선택하겠습니다.

# [Step 3] 변수 선택

# 분석에 활용할 열(속성)을 선택합니다.
ndf = rdf[['survived', 'pclass', 'sex', 'age', 'sibsp', 'parch', 'embarked']]

print(ndf['sex'].unique())

print("ndf['embarked'].unique()")
print(ndf['embarked'].unique())

# 원핫 인코딩을 사용하여 범주형 데이터를 모형이 인식할 수 있도록 숫자형으로 변환해 줍니다.
# 이후 female, male 컬럼이 신규로 생성이 됩니다.
'''get_dummies() 함수는 범주형 데이터를 원-핫 인코딩으로 변환합니다.
예를 들어, ndf['sex']가 "male"과 "female"이라는 두 개의 범주 값을 가진다면, get_dummies()는 두 개의 새로운 열을 생성합니다(각각 "male"과 "female").
결과적으로 onehot_sex에는 각 범주에 대해 1과 0의 값을 가지는 데이터프레임이 생성됩니다. 예를 들어, "male"은 [1, 0], "female"은 [0, 1]로 표현됩니다.
python
'''
print('\nbefore ndf.columns')
print(ndf.columns)

onehot_embarked = pd.get_dummies(data=ndf['embarked'], prefix='town')

ndf = pd.concat([ndf, onehot_sex, onehot_embarked], axis=1)

print('\nafter ndf.columns')
print(ndf.columns)

# 원핫 인코딩에 사용된 컬럼을 제거합니다.

print(x[:5])
```

#### \# ROC 커브 (머신 러닝 교안(이론).pdf (P. 208))
- TPR(True Positive Rate, 민감도/재현율, Sensitivity)
	→ 실제 양성(암) 중에서 모델이 양성으로 맞춘 비율
	$$
TPR = \frac{TP}{TP + FN}
$$
- FPR(False Positive Rate, 거짓 양성률)
	→ 실제 음성(정상) 중에서 모델이 양성이라고 잘못 예측한 비율
	(특이도 Specificity = TN/(TN+FP) 이고, FPR = 1 − Specificity)
	$$
FPR = \frac{FP}{FP + TN}
$$
- 트레이드 오프(Trade-off)
	성능 지표 하나의 값을 높이면 다른 지표의 값이 떨어지는 관계를 의미합니다.
	ROC에서는 재현율(민감도)과 FPR(거짓 양성률) 사이에서 나타납니다.

- 임계값(threshold)
	암이다/아니다를 바로 말하는 게 아니라, 어느 정도 확률부터 암이라고 판정할지 기준선을 정해야 함
	이 기준선이 바로 임계값(Threshold)

- 트레이드 오프에 임계값(threshold)의 관계
	모델은 단순히 "Yes/No"로 예측하는 게 아니라, 실제로 각각의 샘플이 대하여 확률(점수) 값을 먼저 계산하는 방식을 취함
	
| 환자 | 확률 | 임계값(0.3) | 임계값(0.5) | 임계값(0.7) |
| :--- | :--: | :---------: | :---------: | :---------: |
| 김철수 | 92% | yes | yes | yes |
| 박영희 | 65% | yes | yes | no |
| 이순자 | 30% | yes | no | no |
| **특징** | - | 민감함 | 정상 | 보수적 |
Threshold ↓ → 민감해짐 → 환자 잘 잡지만 정상인도 많이 오진 (재현율↑, 정밀도↓)
Threshold ↑ → 보수적 → 정상인 잘 거르지만 환자 놓침 (재현율↓, 정밀도↑)

- ROC 커브 개념
	모델의 판정 기준(임계값)을 바꿔 가며 '민감도(재현율, TPR)'와 '거짓 양성률(FPR)'의 변화를 그린 그래프
	곡선은 x축에 FPR, y축에 TPR(재현 율)을 그려, 분류 임계값에 따른 모델의 성능 변화를 시각화
	FPR이 0이고, TPR이 1의 값을 가지는 왼쪽 상단의 모서리(0, 1)에 가깝게 그려지 는 것이 가장 이상적인 완벽한 모델이라고 볼 수 있음

- AUC
	AUC는 면적 값의 범위가 0.0과 1.0사이의 ROC 곡선 아래의 면적 값을 의미
	값이 1(즉 100%)에 가까울수록 모델이 더 잘 작동한다고 볼 수 있음

#### \# KNN (머신 러닝 교안(이론).pdf (P. 232~))
- 개념
새로운 데이터가 들어오면 기존 데이터 중 가장 가까운 K개의 이웃을 찾아 결과를 예측하는 알고리즘
(지도 학습 - 분류(Classification))


- 동작 원리
```text
Step 1 새로운 데이터 입력

Step 2 기존 데이터와 거리 계산

Step 3 가장 가까운 K개 선택

Step 4 다수결 투표

Step 5 결과 예측
```


- K 값
1) 작은 K
	데이터에 민감하여 복잡한 경계 생성
	-> 과적합(Overfitting) 발생 가능

2) 큰 K
	안정적이지만 세부 패턴을 놓칠 수 있음


- K 값 결정 절차
```text
K 값의 초기 값은 sqrt(n)으로 시작합니다.
전체 개수에 root 값을 씌운 다음 올림/버림을 이용하여 홀수의 개수로 맞춰 줍니다.

하지만 데이터들의 분포(분산)에 따라서 이러한 방법이 항상 최적의 결과를 만들지 못할 수도 있습니다.
이러한 경우 다양한 테스트 데이터 셋에 대하여 일부 K 값을 테스트하여 최적의 분류 성능을 내는 K 값을 선택해 내어야 합니다.
이때 일반적으로 사용하는 방법은 교차 검증(Cross Validation)을 사용하여 가장 성능이 좋은 값을 선택합니다.
K=1, 3, 5, 7, 9, … 등 여러 값을 시도합니다.
각 K에 대한 정확도(Accuracy) 또는 F1 Score를 측정합니다.
가장 좋은 성능을 보이는 K를 선택합니다.

너무 크거나 작은 K는 피하도록 합니다.
홀수 K를 사용하는 경우가 많습니다.
```

- 홀수 K를 사용하는 이유
	분류 문제에서는 다수결(Voting)로 클래스를 결정하므로 홀수 K를 사용하는 경우가 많음
	짝수 K를 사용하면 두 클래스의 표 수가 같아져 동점(Tie)이 발생할 수 있기 때문임

- 장단점
1) 장점
	구현 쉬움
	학습 속도 빠름
	비선형 데이터 처리 가능
	이해하기 쉬움

2) 단점
	데이터가 많으면 모든 데이터와의 거리 계산으로 인하여 속도가 느림
	특성이 많아질수록 성능 저하가 많음(차원의 저주)
	이상치 영향에 영향을 많이 받음



#### \# 과적합과 일반화 (머신 러닝 교안(이론).pdf (P. 78~86))
- 과적합(Overfitting)
1) 개념
	훈련 데이터를 지나치게 학습하여 훈련 데이터에는 높은 성능을 보이지만,
	새로운 데이터에서는 성능이 떨어지는 현상

2) 특징
	훈련 데이터에 지나치게 맞춰 학습함
	데이터의 노이즈(Noise)까지 학습함
	훈련 데이터의 정확도는 높지만 테스트 데이터의 정확도는 낮아질 수 있음
	
3) 예시
	사과 사진이 모두 빨간색이라면 모델이 '빨간색 = 사과'라고 학습할 수 있음.
	이후 초록 사과가 입력되면 사과임에도 불구하고 사과가 아니라고 예측할 수 있음.

4) 그래프로 예시
	산점도의 모든 점을 하나하나 지나가도록 매우 복잡한 곡선을 만들면 과적합될 가능성이 높음.
	데이터의 전체적인 경향(추세)을 잘 나타내는 단순한 곡선일수록 일반화 성능이 좋은 모델이 될 가능성이 높음.

5) 과적합을 막기 위한 사전 점검 항목
	시각화
	기초 통계량 정보에 대한 확인


- 과적합 발생 원인
1) 과적합(Overfitting)
	훈련 데이터를 지나치게 학습하여 노이즈와 이상치까지 학습한 경우
	
2) 데이터 편향(Data Bias)
	특정 집단의 데이터만 많이 포함되어 다양한 데이터에 대해 일반화하지 못하는 경우
	
3) 올바르지 않은 데이터 수집
	특정 집단에 치우치지 않도록 균형 있게 데이터를 수집해야 함
	
4) 오류가 많은 데이터
	결측치, 이상치, 잘못된 값 등이 많으면 모델의 성능과 일반화 능력이 떨어짐

```python
plt.ylabel('Actual label')

knn_report = metrics.classification_report(y_test, prediction)
```

#### \# 의사 결정 트리 (머신 러닝 교안(이론).pdf (P.239))
- 트리(Tree)
	정보를 나무처럼 가지를 뻗어 가며 표현하는 자료 구조
	의사 결정 트리는 계층적으로 의사 결정 규칙들을 트리 형태로 구성하여 데이터를 분석하고 판단하는 모델
	스무 고개 하듯 예/아니오 질문에서 하나를 선택하는 기법
	
	ex)
```text
	여자인가요?
	├── 예
	│   └── 그룹인가요?
	│       ├── 예 → 서현
	│       └── 아니오 → 이미자
	│
	└── 아니오
	    └── 가수인가요?
	        ├── 예 → 김희철
	        └── 아니오 → 유재석
```


- 노드
	트리를 구성하는 각각의 점(지점)
	ex)
		여자인가요? : 최상위 노드인 root 노드
		그룹인가요? : 중간에 추가 질문을 하는 Internal 노드
		서현 : 최종 결과를 나타내는 노드인 잎 노드 (leaf node) (우리가 원하는 solution)
	논리타입으로 양자택일 하는 느낌


- 불순도(Impurity)
	한개의 노드(Node) 안에 서로 다른 클래스가 얼마나 섞여 있는지를 나타내는 정도
	데이터를 잘 분할하면 불순도가 낮아지고, 잘 분할하지 못하면 불순도가 높아짐
	1) 낮은 불순도
		모든 데이터가 같은 종류
		매우 순수(Pure)하다고 표현, 불순도가 0에 가까움
	2) 높은 불순도
		두 종류의 데이터가 섞여 있음
		어떤 클래스인지 구분하기 어렵고, 불순도가 높음


- 엔트로피(Entropy)
(https://www.sciencedirect.com/topics/engineering/shannon-entropy)
데이터가 얼마나 섞여 있는지(불확실성, 불순도)를 나타내는 값
정보의 기대값

1) 높은 엔트로피
	데이터 집합에 서로 다른 종류(클래스)들이 많이 섞여 있음

2) 낮은 엔트로피
	같은 종류(클래스)들이 많이 있음

3) 공식
	기대 값 = $-∑p(xi)*log2p(xi)$


- 정보 이득(Information Gain)
	어떤 질문(분할)을 했을 때 엔트로피가 얼마나 감소했는가를 나타내는 값

1) 공식
	정보 이득 = 분할 전 엔트로피 - 분할 후 엔트로피


- 앙상블 기법
	여러 명의 전문가에 물어 보고 다수의 의견으로 결정을 내리는 방법
	ex) "일대백"이라는 프로그램

1) 목표
	단일 모델보다 여러 모델을 조합하여 더 좋은 예측 성능을 얻는 것

2) 예시
