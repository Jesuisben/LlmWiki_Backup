#### \# CBOW (머신 러닝 교안(이론).pdf (P. 323~324))
- 개념
주변 단어(Context)를 보고 가운데 단어(Target)를 예측하는 방법
결국 패턴을 찾아서 예측하는 방법

- 입력 데이터와 출력 데이터
입력 : 문장
출력 : 가운데 단어 예측측

- 예시
```text
- 예시 문장
  상암 월드컵 경기장에서 우리는 ( A )를 관람합니다.
  
- A의 주변 단어
  상암
  월드컵
  경기장
  우리
  관람
  
- A에 들어갈 후보 단어
  인디아나 존스
  축구
  야구
  봅슬레이

- 초기 상태 확률
  P(인디아나 존스) = 0.25
  P(축구) = 0.25
  P(야구) = 0.25
  P(봅슬레이) = 0.25
  
- 지속적인 학습
  상암 월드컵 경기장에서 축구를 관람했다.
  한국 대표팀의 축구 경기가 열렸다.
  월드컵 축구 경기를 응원했다.
  축구 팬들이 경기장을 가득 메웠다.
  
- 패턴 발견
  월드컵
  경기장
  관람
  대표팀
  응원
  경기
  ->
  월드컵 **축구**
  **축구** 경기장
  **축구** 관람
  **축구** 대표팀
  **축구** 응원
  **축구** 경기

- 확률의 변경
  P(축구)=0.70
  P(야구)=0.20
  P(인디아나 존스)=0.08
  P(봅슬레이)=0.02
  
  P(축구) > P(야구) > P(인디아나 존스) > P(봅슬레이)
```



#### \# Word2Vec (머신 러닝 교안(이론).pdf (P. 325~326))
- 개념
단어를 의미로 표현하기 (단어(Word)를 벡터(Vector)로 변환)
컴퓨터는 글자를 이해하지 못하므로 숫자(벡터)로 변환해야 하는 데, 이를 Word2Vec이라고 함
("같은 문맥(Context)에서 자주 등장하는 단어는 의미가 비슷하다." 라는 아이디어를 이용))

- 단어를 그냥 정수로 변환하지 않고 벡터로 변환하는 이유
정수 한개에는 여러가지 정보를 넣을 수 없는데
단어 하나, 혹은 여러개는 관계성을 가지고 여러가지 특징을 가짐
King과 Queen의 유사도, 상반성 등을 표현하려면 비슷하게 벡터를 구현하는데
정수는 이게 불가능함

- 입력 데이터와 출력 데이터
입력 : 문장
출력 : 단어 벡터(숫자)

- 예시
```text
- 단어 → 변환
  King → [0.45, 0.19, 0.85, ...]
  Queen → [0.44, 0.20, 0.89, ...]
  Man → [0.45, 0.21, 0.88, ...]
  Woman → [0.44, 0.22, 0.87, ...]

- 산술 연산
  연산 : King - Man + Woman
  값 : Queen
-> 이유 : "Word2Vec"으로 단어의 의미(남성, 여성 등)를 벡터(0.45, 0.44 등)로 표현하고
단어를 서로 비교할때 그 벡터(숫자)를 연산해서 결과를 내놓음
King에서 남성(Man)의 의미를 빼고 여성(Woman)의 의미를 더하면 Queen의 의미와 가까워지기 때문.

-> 풀이 :
King - Man + Woman
= (왕 + 남성) - 남성 + 여성
= 왕 + 여성
= Queen

- 특징
  Word2Vec은 의미가 비슷한 단어를 가까운 위치에 배치합니다.
  Word2Vec은 단어 의미를 학습하고, 이를 반영시킵니다.
  계산 효율이 우수합니다.
  
- 시각화
  
- 활용
  검색 엔진
  추천 시스템
  챗봇
  문서 분류
```



#### \# 단어 조각(N-gram) (머신 러닝 교안(이론).pdf (P. 328))
- 개념
N-gram은 문장을 N개의 연속된 단어 묶음으로 만드는 방법
ngram_range=(a, b) : a개부터 b개까지의 모든 n-gram을 생성한다.
ex) ngram_range=(1, 3) 이라면  
1개짜리(Unigram) / 2개짜리(Bigram) / 3개짜리(Trigram) n-gram을 생성함

- 입력 데이터와 출력 데이터
입력 : 문장(또는 문서)
출력 : N-gram(연속된 단어 묶음)

- 단어 예시
I love machine learning


- 항목별 비교

| 항목 | 설명 | 결과 |
|------|------|------|
| Unigram | 단어 1개씩 | I, love, machine, learning |
| Bigram | 단어 2개씩 | I love, love machine, machine learning |
| Trigram | 단어 3개씩 | I love machine, love machine learning |

- 단어 예시
```text
ngram_range=(2, 2) 결과
['love machine' 'machine learning'] # ['I', 'love']에서 'I'는 1글자라서 추출되지 않습니다.

ngram_range=(1, 2) 결과
['bad' 'good' 'not' 'not bad' 'not good' 'very' 'very good']
```

- N-gram을 사용하는 이유
```text
- 단어 예시
  "not good"은 긍정인가요 ?부정인가요?
  
  "good"은 긍정
  "not good"은 부정
  
- Unigram 사용
  not : 부정
  good : 긍정
  
  그럼 "not good"은 긍정이라고 실수할 수 있습니다.
  
- 결론
  N-gram은 여러 단어를 묶어서 문맥(Context)을 반영하기 위한 기법임
```



#### \# TF-IDF 인코딩 (머신 러닝 교안(이론).pdf (P. 337))
- TF-IDF (Term Frequency-Inverse Document Frequency)
TF-IDF 기반으로 문서의 단어 목록들을 행렬 형태로 변환해줌
모든 문서에 공통적으로 들어있는 단어의 경우 문서 구별 능력이 떨어진다고 판단
단어의 개수가 많이 언급된 문서에는 가중치(weight)를 적게 주는 방법

빈도수가 많은 것은 가중치를 낮추고
빈도수가 낮은 것은 가중치를 높임

CountVectorizer + 가중치 정보 = TfidfVectorizer

| 용어 | 설명 |
|------|------|
| TF(Term Frequency) | 특정한 단어가 하나의 문서 안에서 등장하는 빈도 수를 의미합니다. 일반적으로 '조사나 '지시 대명사' 등은 높은 값을 가집니다. |
| DF(Document Frequency) | 문서의 빈도 값으로, 특정 단어가 여러 문서에서 얼마나 자주 등장하는 지를 알려 주는 역할을 합니다. |
| IDF(Inverse DF) | 산술식은 IDF로써, 특정 단어가 다른 문서에 많이 등장하지 않을 수록 값이 커집니다. |

- 입력 데이터와 출력 데이터
입력 : 문장(또는 문서)
출력 : TF-IDF 가중치 벡터

```python
from sklearn.feature_extraction.text import CountVectorizer  
  
texts = [  
    "I love machine learning"  
]

    # 분석할 때 모든 단어들을 2개씩 묶어서 분석해라.  
    # n-gram : 문장 안의 연속된 단어들을 n개씩 묶어 하나의 단어처럼 분석하는 기법

    # -> a개부터 b개까지의 모든 n-gram을 생성한다.  
    # ex) ngram_range=(1, 3) 이라면  
    # 1개짜리(Unigram) / 2개짜리(Bigram) / 3개짜리(Trigram) n-gram을 생성함  
    ngram_range=(2, 2)  
)  
  
X = vectorizer.fit_transform(texts)  
  
print('ngram_range=(2, 2) 결과')  
print(vectorizer.get_feature_names_out())

    "not bad"
```

```python
'''  
2. MLP (다층 퍼셉트론)  
MLP는 가장 기본적인 신경망입니다.  
  
예제:  
good movie  -> 긍정  
bad movie   -> 부정  
  
분류하기  
'''  
  
from sklearn.feature_extraction.text import CountVectorizer  
from sklearn.neural_network import MLPClassifier  
  
import pandas as pd  
  
# "좋은 영화인지 나쁜 영화인지 문장을 보고 판단하는 간단한 AI를 만드는 예제"  
# 데이터  
# input 데이터  
texts = [  
    "good movie",  
    "great movie",  
    "bad movie",  
    "terrible movie"  
]  
  
labels = [1, 1, 0, 0] # 정답 데이터입니다.  
  
# 문장을 숫자로 변환  
# MLP(MLPClassifier)라는 감성 분석를 수행하는  
# 머신러닝 모델에 입력 데이터로 넣기위해서 숫자로 변환함  
vectorizer = CountVectorizer()  
# fit_transform()  
# -> 내부적으로 아래와 같은 단어 사전 생성, 숫자로 변환하는 과정을 거침  
# vectorizer.fit(texts)       # 단어 사전 생성  
# X = vectorizer.transform(texts)  # 숫자로 변환  
X = vectorizer.fit_transform(texts)  
  
print('X')  
print(X)  
  
  
# 단어 목록  
words = vectorizer.get_feature_names_out()  
  
# DataFrame 생성  
df = pd.DataFrame(  
    X.toarray(),  
    columns=words,  
    index=texts  
)  
  
print(df)  
'''  
각 문장을 숫자로 표현  
  
문장  good   great  bad    terrible   moviegood movie  1  0  0  0  1  
great movie 0  1  0  0  1  
bad movie   0  0  1  0  1  
terrible movie  0  0  0  1  1  
'''  
  
# MLPClassifier : # MLP(Multi-Layer Perceptron) 알고리즘입니다.  
# 입력값을 받아 패턴을 학습하고 결과를 예측하는 인공신경망입니다.  
# 입력값은 CountVectorizer가 변환(출력)해준 벡터값  
model = MLPClassifier(  
    hidden_layer_sizes=(5,), # 은닉층(Hidden Layer)의 뉴런 수입니다.  
    max_iter=2000 # 최대 학습 횟수입니다.  
)  
  
model.fit(X, labels)  
  
# 예측  
y_test = "good film"  
test = vectorizer.transform([y_test])  
  
print(f'테스트 데이터 :{y_test}')  
prediction = model.predict(test)  
result = "Positive" if prediction == 1 else "Negative"  
print(f'예측 값 : {result}')
```

- 실제 행렬 일부

- 실행 결과

```text
excellent acting and great storyline -> Positive
```

- 코드 내용
```python
from sklearn.feature_extraction.text import CountVectorizer  
from sklearn.neural_network import MLPClassifier  
  
import pandas as pd  
  
# pd.set_option('display.max_columns', None)  
  
dataIn = '../dataIn/'  
# CSV 읽기  
df = pd.read_csv(dataIn + "movie_review.csv")  
print(df.head())  
  
documents = df["text"] # 모의고사 원본  
y_train = df["label"] # 정답  
  
# 문장을 숫자로 변환 (BoW)vectorizer = CountVectorizer()  
x_train = vectorizer.fit_transform(documents)  
  
# 단어 목록  
words = vectorizer.get_feature_names_out()  
  
# DataFrame 생성  
df = pd.DataFrame(  
    x_train.toarray(),  
    columns=words,  
    index=documents  
)  
  
print("df")  
print(df)  
  
# MLP 모델  
model = MLPClassifier(  
    hidden_layer_sizes=(16,),  
    max_iter=2000,  
    random_state=42  
)  
  
# 학습  
model.fit(x_train, y_train)  
  
# 테스트 문장 5개  
# 신규 데이터  
test_documents = [  
    "this movie was fantastic",  
    "i really loved this film",  
    "the story was boring",  
    "this was the worst movie ever",  
    "excellent acting and great storyline"  
]  
  
# 학습한 MLP 모델에 새로운 데이터를 넣기 위해 벡터로 변환함  
x_test = vectorizer.transform(test_documents)  
  
# 학습한 MLP 모델에 벡터화한 새로운 데이터를 넣어 예측하기  
# 반환타입은 numpy.ndarray임  
predictions = model.predict(x_test)  
  
for sentence, pred in zip(test_documents, predictions):  
    result = "Positive" if pred == 1 else "Negative"  
    print(f"{sentence} -> {result}")
```
