#### \# Terraform 설명 (cloud.03.AWS 교안(이론).pdf(P.232))
- 인프라 : 컴퓨터와 네트워크 환경에서 서비스를 운영하기 위해 필요한 기반 환경 전체
소프트웨어가 돌아가기 위해 필요한 "하드웨어, 네트워크, 서버, 스토리지, 데이터베이스, 로드 밸런서, 보안 설정 등" 모든 것을 포함하는 개념
ex) Terraform - 인프라 자동화를 대표하는 도구
-\> 여러 클라우드 서비스(AWS, Azure, GCP, OCI 등)를 HCL라는 단일 언어로 제어가능

- Terraform 구성 요소
Provider / Resource / Module / Variable / TerraFrom
Module : 비슷한 것들을 묶은 것 (ex-상품모듈, 회원모듈 등)

- HCL
Terraform의 핵심 언어
사람이 읽기 쉽고, 기계도 실행 가능한 선언적 문법
JSON처럼 구조화된 구성을 제공해주며, 확장자는 ".tf" 형식으로 작성

- HCL 사용 방법
중괄호 기호를 사용하여 블록 구조 기반 형식으로 사용
ex)
```hcl
provider "aws" {
	region = "ap-northeast-2"
}

resource "aws_instance" "myserver" {
	ami = "ami-0abcd1234abcd1234"
	instance_type = "t2.micro"
	tags = {
		Name = "WebServer"
	}
}
```

- Terraform 실습때 사용하는 코드 (cmd에 입력)
1) terraform init
Terraform 작업 디렉터리 초기화
프로바이더(provider) 다운로드

2) terraform plan
실제 인프라 적용전 어떤 리소스가 생성, 변경, 삭제될 것인지 미리 확인
사전 점검의 개념

3) terraform apply
작성한 설정 파일을 기반으로 실제 클라우드 환경에 리소스 생성/적용
정의된 인프라가 실제로 배포됨

4) terraform destroy
코드에 정의된 리소스 삭제 기능
terraform destroy -target=some_resource와 같이 특정 리소스만 삭제 가능
