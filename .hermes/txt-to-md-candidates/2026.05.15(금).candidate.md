2. Passwordless X1280 Docker 컨테이너 실행
1) Docker hub (설치 스크립트 및 설정 참조) :
https://hub.docker.com/r/dualauth/passwordlessx1280-single
Docker hub 내 Guide의 안내에 따라 컨테이너를 실행

실행이 완료되면
- Auth (인증) 서버
- User Connection (유저커넥션) 서버
- Push Request (푸시 요청) 서버
의 Passwordless X1280 서버 3 종이 자동으로 설치완료

“-e USE_SSL=false” 옵션 사용 시 SSL 사용 안함

\* 명령어
-\>
```shell
docker run -d --name server \
--network some-network \
--restart always \
-e DOMAIN=Passwordless서버의 도메인 \
-e USE_SSL=false \
-v auth-settings:/opt/x1280/tomcat/conf/passwordless \
-v auth-logs:/opt/x1280/tomcat/logs \
-v user-connection-logs:/opt/x1280/connector/logs \
-v push-request-logs:/opt/x1280/pushConnector/logs \
-v config:/etc/opt/x1280 \
-v database:/var/lib/mysql \
-p 8005:8005 \
-p 8080:8080 \
-p 8180:8180 \
-p 8143:8143 \
-p 8443:8443 \
-p 11040:11040 \
-p 12010:12010 \
-p 15010:15010 \
dualauth/passwordless-x1280-single:latest
```

2) 방화벽 Open (Rocky Linux)
Docker 컨테이너의 방화벽이 아니고 Docker가 설치된 Host 서버의 방화벽을 오픈
Docker 컨테이너의 방화벽은 Docker 컨테이너 설치 시 자동으로 오픈됨

```shell
[root@localhost ~]# dnf install firewalld

[root@localhost ~]# firewall-cmd --zone=public --permanent --add-port=8005/tcp
[root@localhost ~]# firewall-cmd --zone=public --permanent --add-port=8080/tcp
[root@localhost ~]# firewall-cmd --zone=public --permanent --add-port=8180/tcp
[root@localhost ~]# firewall-cmd --zone=public --permanent --add-port=8143/tcp
[root@localhost ~]# firewall-cmd --zone=public --permanent --add-port=8443/tcp
[root@localhost ~]# firewall-cmd --zone=public --permanent --add-port=11040/tcp
[root@localhost ~]# firewall-cmd --zone=public --permanent --add-port=12010/tcp
[root@localhost ~]# firewall-cmd --zone=public --permanent --add-port=15010/tcp
```

3-1. SSL 인증서 적용
1) 인증서 미사용시
Docker container 설치 시 “-e USE_SSL=false” 옵션 사용

2) 인증서 사용시
2-1) Passwordless서버의 도메인용으로 발급된 인증서를 준비
- cert.pem
- privkey.pem

2-2) 발급된 인증서를 이용하여 Tomcat keystore 파일 생성

2-3) Docker container 설치 시 -e USE_SSL=false 옵션 대신
아래 내용을 추가하여 실행
```shell
-e AUTH_KEYSTORE_FILE_PATH=/etc/opt/x1280/ssl/tomcat.keystore \
-e AUTH_KEYSTORE_PASS= keystore 생성시 적용한 비밀번호 \
-e SSL_CERT_PATH=/etc/opt/x1280/ssl/cert.pem \
-e SSL_CERT_KEY_PATH=/etc/opt/x1280/ssl/privkey.pem \
```

2-4) Docker container ID 찾기
```shell
docker container ps -a
```

2-5) Docker Container 내부에 ssl 폴더 생성
```shell
docker exec 87d5679c7f90 mkdir /etc/opt/x1280
docker exec 87d5679c7f90 mkdir /etc/opt/x1280/ssl
```

2-6) Docker Container 내부로 인증서파일 복사
(/home/ssl/ 폴더는 docker container 내부가 아니고 docker host서버의 폴더임)
```shell
docker cp /home/ssl/cert.pem 87d5679c7f90:/etc/opt/x1280/ssl
docker cp /home/ssl/privkey.pem 87d5679c7f90:/etc/opt/x1280/ssl
docker cp /home/ssl/tomcat.keystore 87d5679c7f90:/etc/opt/x1280/ssl
```

2-7) Docker container 재기동
```shell
docker restart server
```
